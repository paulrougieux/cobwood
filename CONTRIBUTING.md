# Contributing to cobwood

Thank you for your interest in contributing to cobwood! We welcome contributions from
the community.


## Making Changes

1. Create a new branch for your work:
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. Make your changes and format your code with Black:
   ```bash
   black .
   ```

3. Run tests to ensure everything works:
   ```bash
   python -m pytest  # or your project's test command
   ```

4. Commit your changes with clear messages:
   ```bash
   git commit -m "Add feature: brief description"
   ```

## Submitting a Merge Request

1. Push your changes to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```

2. Open a Merge Request on GitLab with:
   - Clear description of your changes
   - Reference to any related issues
   - Confirmation that tests pass


## Code Style

- Use the **Black** code formatter on Python code
- Follow existing code conventions
- Write clear commit messages using present tense ("Add feature" not "Added feature")


## Reporting Issues

When reporting bugs or suggesting features, please include:
- Clear description of the issue or enhancement
- Steps to reproduce (for bugs)
- Expected vs actual behavior
